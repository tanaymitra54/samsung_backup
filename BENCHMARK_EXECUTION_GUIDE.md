Each question is evaluated with 3 methods:

### 1. Greedy (Baseline)
- Direct question → answer
- No chain-of-thought
- Deterministic
- Fastest (~0.5s per question)

### 2. Chain-of-Thought (CoT)
- Prompt: "Let's think step by step..."
- Single reasoning trace
- Deterministic
- Fast (~1s per question)

### 3. QUBO Pipeline (Full)
- 16 diverse reasoning traces
- Correctness scoring
- QUBO optimization
- Final answer synthesis
- Slow (~9s per question)

## Troubleshooting

### Issue: Out of Memory (OOM)

**Symptoms:** `RuntimeError: CUDA out of memory`

**Solutions:**
```bash
# Option 1: Enable 4-bit quantization
# Edit config/config.yaml:
# model.load_in_4bit: true

# Option 2: Reduce batch size
# Edit config/config.yaml:
# evaluation.batch_size: 2  # or 1

# Option 3: Reduce parallel reads
# Edit config/config.yaml:
# solver.gpu.num_parallel_reads: 512
```

### Issue: Very Slow Execution

**Symptoms:** 20+ seconds per question

**Solutions:**
```bash
# Reduce sampling diversity
# Edit config/config.yaml:
# pipeline.num_answers: 2
# pipeline.num_reasons: 2
# (Results in 8 samples instead of 16)

# Enable vLLM (if installed)
# Edit config/config.yaml:
# model.use_vllm: true
```

### Issue: Low QUBO Accuracy

**Symptoms:** QUBO worse than Greedy/CoT

**Investigations:**
1. Check verifier scores in sampling
2. Verify QUBO matrix has reasonable values
3. Look at selected vs. rejected traces
4. Check if final prompt is well-formed

### Issue: Benchmark Fails to Load

**Symptoms:** `Unknown benchmark` or `NotImplementedError`

**Solutions:**
- Remove failing benchmark from command
- For StrategyQA: Skip it (known issue in codebase)
